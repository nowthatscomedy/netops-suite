# Switch Config Builder 처음 사용자 튜토리얼

이 문서는 이 프로그램을 처음 실행하는 사용자가 샘플 파일로 바로 CLI를 만들어 보고, 자기 환경에 맞게 수정하는 데 필요한 최소 흐름을 안내합니다.

## 이 프로그램이 하는 일

`Switch Config Builder`는 아래 두 가지를 조합해서 스위치 초기 설정 CLI를 만들어 줍니다.

1. `profiles/` 안의 YAML 템플릿
2. `device_values/` 안의 CSV/XLSX 장비 정보 파일

쉽게 말하면:

- 템플릿은 "어떤 명령을 어떤 순서로 출력할지"를 정의합니다.
- 장비 파일은 "각 장비의 hostname, IP, VLAN, 비밀번호 같은 실제 값"을 담습니다.
- 프로그램은 이 둘을 합쳐서 장비별 CLI를 오른쪽 패널에 바로 보여 줍니다.

## 이 튜토리얼에서 사용할 파일

처음에는 아래 두 파일로 연습하는 것이 가장 쉽습니다.

- `profiles/sample_comprehensive_reference.yaml`
- `device_values/sample_comprehensive_reference_devices.csv`

이 샘플은 정상 예시와 오류 예시를 함께 담고 있습니다. 그래서 처음 열었을 때 일부 행이 `확인 필요`로 보이는 것이 정상입니다.

추가로 샘플 CSV의 아래 컬럼은 학습용 참고 컬럼입니다.

- `sample_case`
- `expected_result`
- `note`

이 컬럼들은 CLI 생성에 꼭 필요하지 않으며, 템플릿에 없는 컬럼은 렌더링 때 무시됩니다.

## 시작 전에 꼭 알아둘 점

1. `profiles/`의 템플릿은 프로그램 시작 시 자동으로 불러옵니다.
2. 장비 데이터는 CSV 또는 XLSX 파일을 직접 열어서 편집합니다.
3. `실시간 파일 저장`이 기본으로 켜져 있습니다.
4. 샘플 파일을 보존하고 싶다면, 편집하기 전에 파일을 복사하거나 `실시간 파일 저장`을 먼저 꺼 두는 것이 안전합니다.
5. `profile_id` 값은 템플릿의 `id`와 정확히 같아야 합니다.
6. `device_id`는 장비 식별용으로 넣는 것을 권장하지만, 현재 동작상 필수값은 아닙니다.

## 1. 프로그램 실행하기

프로젝트 루트에서 아래 명령을 실행합니다.

```bash
python -m pip install -r requirements.txt
python desktop_app.py
```

또는:

```bash
python app.py
```

## 2. 화면을 한 번에 이해하기

처음 보면 버튼이 많아 보이지만, 실제로는 아래 흐름만 기억하면 됩니다.

1. 상단 `기준 템플릿`
선택한 템플릿의 설명, 블록 목록, 새 행 추가 기준이 됩니다.

2. 상단 `명령 블록 선택`
선택한 템플릿의 블록을 통째로 켜거나 끕니다.

3. 상단 `필터`
특정 값만 빠르게 찾습니다. 샘플 파일에서는 `expected_result = 생성 가능`만 보도록 필터링할 때 특히 유용합니다.

4. 상단 `행 작업`
행 추가, 선택 행 복사, 연속 값 복사, 삭제를 합니다.

5. 상단 `파일 상태`
데이터 파일 열기, 현재 파일 경로, 자동 저장 상태를 보여 줍니다.

6. 왼쪽 큰 표
장비 정보 자체를 편집하는 곳입니다. 엑셀처럼 값을 붙여 넣을 수 있습니다.

7. 오른쪽 패널
선택한 장비의 요약과 `안내`, `변수`, `이슈`, `CLI` 탭이 표시됩니다.

## 3. 샘플 파일로 첫 CLI 만들기

가장 먼저 이 흐름을 그대로 따라 해 보세요.

1. 프로그램을 실행합니다.
2. 상단 `데이터 열기`를 눌러 `device_values/sample_comprehensive_reference_devices.csv`를 엽니다.
3. 상단 `기준 템플릿`에서 `SAMPLE_COMPREHENSIVE_REFERENCE`를 선택합니다.
4. 샘플에는 오류 행도 섞여 있으므로, 상단 `필터 항목`을 `expected_result`로 바꾸고 `필터 값`에 `생성 가능`을 입력합니다.
5. 이제 `SW-REF-01` 같은 행을 하나 클릭합니다.
6. 오른쪽 `안내` 탭에서 템플릿 설명을 읽습니다.
7. 오른쪽 `변수` 탭에서 현재 값, 필수 여부, 기본값을 확인합니다.
8. 오른쪽 `이슈` 탭이 비어 있거나 경고가 없으면 정상입니다.
9. 오른쪽 `CLI` 탭에서 생성된 스위치 설정을 확인합니다.
10. `복사`를 누르면 현재 장비 CLI가 클립보드로 복사됩니다.
11. 여러 대를 순서대로 작업할 때는 `복사+다음`을 쓰면 편합니다.

## 4. 샘플을 내 환경 값으로 바꾸기

이제 표에서 값을 직접 바꿔 보세요.

추천 순서:

1. `hostname`
2. `mgmt_ip`
3. `mgmt_mask`
4. `mgmt_gw`
5. 필요 시 `access_vlan`, `voice_vlan`, `ntp_server` 같은 옵션 값

값을 바꾸면 오른쪽 `CLI`가 다시 계산됩니다.

만약 `CLI`가 사라지거나 상태가 `확인 필요`로 바뀌면, 오른쪽 `이슈` 탭을 먼저 보세요. 보통은 아래 이유입니다.

| 증상 | 흔한 원인 | 해결 방법 |
| --- | --- | --- |
| `profile_id 값이 비어 있습니다.` | 템플릿 ID가 없음 | `profile_id`에 올바른 템플릿 ID를 입력 |
| `profile_id에 해당하는 템플릿을 찾을 수 없습니다.` | 오탈자 또는 없는 템플릿 | `profiles/`에 있는 실제 ID와 동일하게 수정 |
| 필수값 누락 | required 값이 비어 있음 | 해당 컬럼을 채움 |
| `IPv4`, `int`, `bool` 형식 오류 | `192.168.1.300`, `VLAN99`, `maybe` 같은 값 입력 | 형식에 맞는 값으로 수정 |
| `hostname 값이 중복되었습니다.` | 호스트명이 다른 행과 같음 | hostname을 고유하게 변경 |
| `IP 값이 중복되었습니다.` | 관리 IP 등 IP 컬럼이 다른 행과 같음 | IP를 고유하게 변경 |
| `mgmt_gw 값이 mgmt_ip/mgmt_mask 대역과 맞지 않습니다.` | 게이트웨이가 같은 대역이 아님 | IP, 마스크, 게이트웨이 대역을 맞춤 |

참고:

- 비밀번호처럼 `secret`, `password`, `key`가 들어간 컬럼은 표와 변수 안내에서 마스킹되어 보일 수 있습니다.
- 표는 엑셀처럼 여러 셀 복사/붙여넣기가 가능합니다.

## 5. 여러 장비를 빨리 만드는 방법

### 새 행 추가

1. 상단 `기준 템플릿`에서 원하는 템플릿을 선택합니다.
2. `행 추가`를 누릅니다.
3. 선택한 템플릿에 필요한 컬럼이 자동으로 준비된 빈 행이 추가됩니다.

### 선택 행 복사

1. 기존 행을 선택합니다.
2. `선택 행 복사`를 누릅니다.
3. 값이 그대로 복제되므로, hostname과 IP 같은 중복 가능 값은 직접 바꿔야 합니다.

### 연속 값 복사

이 기능은 템플릿에 `auto_increment` 규칙이 들어 있는 변수만 자동 증가시켜 복사합니다.

샘플 템플릿에서는 대표적으로 아래 값이 자동 증가합니다.

- `hostname`
- `mgmt_ip`

사용 방법:

1. 원본 행을 선택합니다.
2. `연속 값 복사`를 누릅니다.
3. 몇 개를 더 만들지 입력합니다.
4. 생성된 행에서 결과를 확인합니다.

중요:

- 현재 기본 동작에서는 `device_id`는 자동 증가 대상이 아닙니다.
- 따라서 `연속 값 복사`를 쓴 뒤에는 필요하면 `device_id`를 직접 수정하세요.

## 6. 블록 on/off와 데이터 값의 차이

처음 사용할 때 가장 헷갈리기 쉬운 부분입니다.

### 상단 `명령 블록 선택`

- 템플릿의 블록 전체를 켜거나 끕니다.
- 예를 들어 `ntp` 블록을 끄면, 그 템플릿을 사용하는 행들의 CLI에서 NTP 관련 명령이 빠집니다.

### 표 안의 true/false 컬럼

- 일부 템플릿은 `enable_voice_vlan` 같은 컬럼을 Jinja 조건문에 사용합니다.
- 이런 경우에는 행 데이터 값에 따라 특정 줄이 출력되거나 빠집니다.

정리하면:

- "블록 전체를 잠깐 빼고 싶다"면 상단 블록 체크박스를 사용합니다.
- "특정 행의 옵션 값으로 세부 출력을 바꾸고 싶다"면 표 안의 컬럼 값을 수정합니다.

## 7. 저장하기

장비 데이터 저장과 CLI 복사는 다른 작업입니다.

### 장비 데이터 파일 저장

- 상단 툴바의 `저장`은 현재 CSV/XLSX를 저장합니다.
- `다른 이름 저장`은 새 파일로 저장합니다.
- 기존 파일을 수동 저장할 때는 `outputs/backups/`에 백업 파일이 만들어집니다.

### 자동 저장

- `실시간 파일 저장`이 켜져 있으면 편집 후 자동으로 현재 파일에 반영됩니다.
- 다만 오류가 있으면 자동 저장은 기본적으로 보류됩니다.
- `오류 있어도 실시간 저장`은 정말 필요할 때만 켜는 것을 권장합니다.

처음에는 이렇게 쓰는 것이 안전합니다.

1. 샘플 파일을 복사해서 새 이름으로 연습한다.
2. 또는 `실시간 파일 저장`을 끄고 익숙해진 뒤 수동 저장한다.

## 8. 내 템플릿 만들기

샘플을 조금 써 본 뒤에는 기존 템플릿을 복사해서 시작하는 것이 가장 쉽습니다.

1. 상단 `기준 템플릿`에서 비슷한 템플릿을 선택합니다.
2. `템플릿 복사`를 누릅니다.
3. 템플릿 ID, 벤더, 모델, 설명을 수정합니다.
4. `변수` 탭에서 필요한 컬럼을 추가하거나 삭제합니다.
5. `명령 블록` 탭에서 CLI 라인을 수정합니다.
6. 저장합니다.
7. 이후 장비 CSV/XLSX의 `profile_id`에 새 템플릿 ID를 넣으면 됩니다.

처음부터 새 템플릿을 완전히 빈 상태로 만드는 것보다, 샘플을 복사해서 필요한 부분만 줄이는 방식이 훨씬 빠릅니다.

## 9. 완전 초보자용 추천 연습 순서

처음 한 번은 아래 순서만 따라 하면 충분합니다.

1. 샘플 CSV를 연다.
2. `expected_result = 생성 가능`만 필터링한다.
3. 한 행을 클릭해서 오른쪽 `CLI`를 확인한다.
4. `hostname`과 `mgmt_ip`를 바꿔 본다.
5. `복사`로 CLI를 가져간다.
6. `선택 행 복사` 또는 `연속 값 복사`로 장비 수를 늘린다.
7. 익숙해지면 템플릿을 복사해서 내 표준 템플릿을 만든다.

## 10. 막히면 먼저 확인할 것

1. `profile_id`가 실제 템플릿 ID와 정확히 같은가
2. 필수값이 비어 있지 않은가
3. IP, 정수, bool 값 형식이 맞는가
4. 다른 행과 hostname 또는 관리 IP가 중복되지 않았는가
5. 오른쪽 `이슈` 탭에 무엇이 적혀 있는가

이 다섯 가지만 확인해도 대부분의 문제는 바로 찾을 수 있습니다.
