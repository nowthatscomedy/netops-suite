# NetOps Suite 릴리즈 노트

공식 릴리즈는 GitHub가 자동 생성한 커밋 목록 대신 이 디렉터리의 검토된 Markdown을 본문으로 사용합니다.

## 작성 방법

1. `TEMPLATE.md`를 복사해 출시 태그와 같은 이름으로 저장합니다. 예: `v1.0.10.md`
2. H1을 `# NetOps Suite v1.0.10` 형식으로 맞춥니다.
3. 템플릿의 모든 `##` 섹션을 실제 내용으로 채웁니다. 해당 사항이 없으면 `해당 없음`이라고 명시합니다.
4. 이전 버전과 비교해 달라진 동작, 새 기능, 수정된 문제와 설치 주의사항을 사용자 관점에서 작성합니다.
5. 자동화 테스트 결과와 제공 자산의 무결성 정보를 확인한 뒤 릴리즈 워크플로를 실행합니다.

릴리즈 워크플로는 대응하는 파일이 없거나, 버전 제목이 다르거나, 필수 섹션이 비어 있거나,
`TODO`·`TBD` 같은 임시 문구가 남아 있으면 게시를 중단합니다.
