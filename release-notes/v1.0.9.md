# NetOps Suite v1.0.9

`v1.0.9`는 `v1.0.8` 이후 실사용 QA에서 확인된 문제를 반영한 기능·사용성·안정성 업데이트입니다.
NetOps 어시스턴트, 연결 진단, 설정 화면, 장비 점검 프로파일 작성 흐름을 집중적으로 개선했습니다.

> [!IMPORTANT]
> 이번 설치 파일은 요청에 따라 코드 서명되지 않은 미서명 설치본입니다. Windows Defender 또는
> SmartScreen에서 게시자를 확인할 수 없다는 경고가 표시될 수 있으므로, 공식 GitHub 릴리즈에서
> 내려받고 아래 SHA-256 값을 확인한 뒤 실행하세요.

## 한눈에 보기

- AI 채팅의 세션 유지, 모델명 표시, Markdown 표, 부분 복사, 파일·이미지 붙여넣기를 개선했습니다.
- Ping과 TCPing을 비롯한 다중 대상 작업이 입력된 모든 대상을 빠짐없이 처리하도록 수정했습니다.
- 설정과 저장 위치를 한 화면 체계로 통합하고 모든 설정 초기화 및 OUI 데이터 업데이트 기능을 추가했습니다.
- 연결 진단, 장비 점검 프로파일, 작은 화면 레이아웃과 작업 상태 표시를 실사용 기준으로 다듬었습니다.
- 해시 고정 의존성, 자동화 QA, 비밀정보 검사, 체크섬과 CycloneDX SBOM을 릴리즈 과정에 추가했습니다.

## 이전 버전과 달라진 점

### NetOps 어시스턴트

- 대화마다 일반적인 `AI` 표시 대신 실제 AI 서비스와 모델명을 표시합니다.
- 같은 서비스의 대화 세션을 기본적으로 이어가며, 필요할 때 `세션 초기화`로 새 세션을 시작할 수 있습니다.
- Markdown 표, 긴 응답, 코드 블록의 표시와 크기 계산을 개선했습니다.
- 메시지 일부를 마우스로 선택해 복사하거나 메시지 전체를 복사할 수 있습니다.
- AI가 응답을 준비하거나 NetOps 기능을 실행하는 동안 작업 상태를 표시합니다.
- 사용자 요청, 기능 실행 안내와 실제 실행 결과가 자연스러운 시간 순서로 표시됩니다.

### 연결 진단

- Ping은 입력한 모든 대상을 실행합니다.
- TCPing은 입력한 모든 `대상 × 포트` 조합을 실행합니다.
- AI 채팅에서 여러 IP, 도메인, MAC 주소 또는 인터페이스를 요청해도 일부만 처리하지 않습니다.
- 빠른 실행 버튼의 크기를 통일하고 중복 버튼을 제거했습니다.
- 횟수와 제한 시간 입력란에는 숫자만 표시하고 `회`, `ms` 같은 단위는 별도 레이블로 분리했습니다.
- 임의의 Ping/TCPing 동시 작업 수 설정을 제거했습니다.

### 설정과 저장 위치

- 별도의 `결과 파일` 화면을 제거하고 설정을 `프로그램`, `저장 위치`, `도구 연동`, `설정 관리`로 정리했습니다.
- 설정·로그·결과/내보내기 폴더를 한곳에서 변경하고 해당 폴더를 바로 열 수 있습니다.
- 중복되어 있던 Ping/TCPing 기본값 설정과 불필요한 내장 TCPing 설명을 제거했습니다.
- CSV, 로그, 진단 결과, 설정 생성 파일과 AI 대화를 저장할 때 위치와 파일명을 직접 선택할 수 있습니다.

## 새 기능

### Codex 권한 선택

- `읽기 전용`: 파일을 읽을 수 있지만 변경하지 않습니다.
- `작업공간 액세스`: 설정, 로그, 결과와 사용자 프로파일 폴더 안에서 작업할 수 있습니다.
- `전체 액세스`: 인터넷과 컴퓨터의 모든 파일에 접근할 수 있으며, 선택 전에 별도 경고와 확인 절차를 거칩니다.

### 채팅 첨부와 대화 관리

- 파일을 채팅 입력창에 붙여넣거나 끌어 놓을 수 있습니다.
- 클립보드 이미지를 바로 붙여넣어 PNG 첨부로 전달할 수 있습니다.
- 각 첨부 카드의 `X` 버튼으로 개별 파일을 제거할 수 있습니다.
- 대화 내용을 Markdown 또는 텍스트 파일로 사용자가 선택한 위치에 저장할 수 있습니다.
- AI 서비스가 제공하는 모델 목록을 새로고침하고 마지막 정상 목록을 캐시합니다.

### OUI 제조사 데이터 관리

- ARP 스캔, Wi-Fi 분석과 MAC 제조사 조회가 함께 사용하는 OUI 데이터의 로컬 버전과 원본 정보를 표시합니다.
- IEEE 공식 데이터와 비교해 최신 여부를 확인할 수 있습니다.
- 필요한 원본을 모두 정상적으로 받은 경우에만 데이터를 교체하며, 실패하면 기존 데이터를 유지합니다.

### 모든 설정 초기화

- 프로그램 옵션, 화면 상태, 네트워크·파일 전송 프로파일, AI 설정과 사용자 장비 점검 규칙을 기본값으로 되돌릴 수 있습니다.
- 기존 애플리케이션 로그, 실행 결과, 백업 파일과 사용자가 내보낸 파일은 삭제하지 않습니다.
- 초기화 후에는 프로그램을 다시 시작해야 합니다.

## 개선 및 수정

### 장비 점검/백업

- 프로파일 작성 화면에서 `점검 명령`과 `백업 명령`을 별도 탭으로 분리했습니다.
- 백업 사용 여부와 백업 명령의 용도를 더 명확하게 표시합니다.
- 빈 백업 명령, 잘못된 정규식과 중복 Excel 컬럼 등을 저장 전에 검사합니다.
- 저장하지 않은 변경이 있을 때 프로파일 전환이나 창 닫기를 확인합니다.
- 외부에서 사용자 프로파일을 변경해도 프로그램 전체를 다시 시작하지 않고 불러올 수 있습니다.
- 실행 중인 장비 점검/백업 작업을 중지할 수 있습니다.
- 결과 Excel은 사용자가 명시적으로 `결과 열기`를 눌렀을 때만 엽니다.

### 화면과 접근성

- 1024×680을 포함한 작은 화면에서도 주요 입력란과 실행 버튼에 접근할 수 있도록 조정했습니다.
- 작은 화면에서는 네트워크 인터페이스 표의 보조 열을 정리해 핵심 정보와 버튼을 유지합니다.
- 키보드로 왼쪽 내비게이션과 현재 화면의 주요 조작부 사이를 이동할 수 있습니다.
- 긴 AI 응답을 읽는 중에는 임의로 맨 아래로 이동하지 않으며, 마지막 내용을 보고 있을 때만 새 응답을 따라갑니다.
- 파일 첨부 카드의 제거 버튼이 카드 테두리 밖으로 벗어나지 않도록 수정했습니다.
- Wi-Fi 조회와 주변 AP 스캔의 작업 중·완료·실패 상태를 명확히 표시합니다.

### 안정성과 장애 진단

- 애플리케이션 로그에 버전, 운영체제, Python/Qt 버전, 관리자 권한, 프로세스·스레드와 코드 위치를 기록합니다.
- 메인 스레드, 작업 스레드와 Qt 오류의 전체 traceback을 장애 로그에 남깁니다.
- 비밀번호, API 키, 인증 토큰과 URL 자격 증명 등 일반적인 민감정보 형태를 로그에서 자동 마스킹합니다.
- 로그 크기와 백업 개수를 제한해 장기간 사용 시 디스크 사용량이 과도하게 증가하지 않도록 했습니다.
- JSON 설정은 임시 파일을 이용해 원자적으로 저장하고, 손상된 설정 파일은 백업한 뒤 복구합니다.
- 작업 시작 실패, 취소와 오래된 비동기 응답이 새 작업에 영향을 주는 문제를 보완했습니다.
- DHCP 전환, 고정 IP·게이트웨이 적용과 PowerShell timeout 오류 처리를 강화했습니다.

## 설치 및 주의사항

- 지원 환경: **64비트 Windows 10 버전 1809(빌드 17763) 이상**
- 이번 설치본은 **코드 서명되지 않은 미서명 설치본**입니다.
- SHA-256은 파일 무결성 확인 수단이며 게시자 신원을 증명하는 코드 서명을 대신하지 않습니다.
- DHCP, 고정 IP, 게이트웨이·DNS 변경 등 일부 작업은 Windows 관리자 권한이 필요합니다.
- `모든 설정 초기화`는 사용자 장비 점검 규칙과 Python 파서를 포함한 설정을 제거하므로 필요한 파일을 먼저 백업하세요.
